# RSP

## 摘要
RSP 是面向 Kerbal Space Program (KSP) 的自动化控制与回收研究型框架，目标是在 RSS + RO 场景中，基于 kRPC 实现多舰体协同控制、自动回收与 AI 辅助决策。系统采用进程级隔离的控制架构，将实时控制、AI 推理与可视化交互解耦，从而满足高频遥测采样与控制回路稳定性的工程需求。

## 研究背景与动机
在 RSS/RO 环境下，航天器动力学强非线性、空气动力学与推进系统参数变化显著，单一控制回路难以覆盖发射、轨道机动与回收等多阶段任务。RSP 以工程可用性为前提，结合经典轨道力学算法与 AI 控制接口，构建一套可扩展的任务管线与自动回收策略，重点面向多舰体分离后的自主识别与并行控制。

## 系统架构
RSP 采用三进程协同结构：

- 控制内核（Kernel）：与 kRPC 建立连接，负责多飞行器实体管理、状态机驱动与遥测采样。
- AI 服务（AI Service）：独立进程执行模型推理或经典控制算法，避免阻塞控制回路。
- 可视化与交互（Dashboard）：展示遥测数据、接收外部指令并转发至内核。

该结构降低了 GIL 对实时控制的影响，并为后续引入更复杂的推理模型提供了可扩展接口。

## 方法与算法
### 1. 轨道力学与机动计算
RSP 内置轨道力学计算模块，提供以下基础能力：

- 活力公式（Vis-viva）用于轨道速度估计
- 霍曼转移（Hohmann Transfer）用于双圆轨道变轨的 Δv 与转移时间计算
- 相位角（Phase Angle）估计用于转移窗口规划

这些算法用于辅助轨道规划与 AI 策略评估的特征构造。

### 2. 自动回收策略
回收流程基于状态机驱动，典型阶段包括下降识别、姿态稳定、自杀燃烧与着陆腿展开。核心控制逻辑包括：

- 下降段逆行姿态控制（RCS/SAS 兜底）
- 自杀燃烧高度估计：基于推重比与重力模型计算刹车点
- 低空阶段控制优先级提升，覆盖 AI 输出的油门建议

该流程支持多舰体并行运行，并在回收成功后自动退出控制。

### 3. AI 接入与控制策略
RSP 提供统一 AI 控制接口：

- 支持接入深度学习模型或经典控制器（PID）
- 采用队列式请求/响应模型，实现控制与推理解耦
- 提供本地控制器作为兜底，确保在模型不可用时维持可控性

AI 接口可用于姿态控制、油门调节、特征驱动的策略选择等任务。

## 代码结构

- sync.py：系统入口，启动 Kernel / AI Service / Dashboard
- core/kernel.py：控制内核，负责实体监测与遥测流
- core/RockerCore.py：单飞行器控制逻辑与回收状态机
- core/orbit.py：轨道力学计算
- core/ai_interface.py：AI 控制接口与 PID 兜底
- core/ai_service.py：独立 AI 推理进程
- utility/：日志、配置、仪表盘与通用工具

## 运行环境

- Python 版本：建议 3.11+
- 依赖组件：kRPC、KSP（RSS + RO）
- 运行方式：使用 uv 管理 Python 环境

## 快速开始

1. 确保 kRPC 已在 KSP 内启动，并确认端口可达。
2. 配置 config.toml，设置任务组映射与参数。
3. 启动系统：

```bash
uv run sync.py
```

## 设计原则与可扩展性
RSP 追求研究可复现与工程可维护性的平衡：

- 控制回路与推理回路隔离，保证实时性
- 状态机与 AI 策略分离，支持多任务扩展
- 模块化设计便于替换算法与接入新模型
